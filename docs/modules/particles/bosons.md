# Bosons

## Overview

Bosons are force-carrying particles with integer spin. Geant4's boson module includes the photon (electromagnetic force carrier), optical photons for optical physics simulations, phonons for condensed matter physics, and special utility particles (geantinos) used for geometry debugging and testing. These particles are essential for electromagnetic radiation, optical processes, and simulation validation.

Unlike fermions, bosons can occupy the same quantum state and do not obey the Pauli exclusion principle. The boson implementations in Geant4 include both physical gauge bosons and simulation-specific particles.

## Particle List

### Physical Bosons

|--------------|---------|----------|------------|---------|------|-------------|

| Optical Photon | γₒₚₜ | -22 | 0 | 0 | 1 | `G4OpticalPhoton` |

| Particle Name | Description | PDG Code | Mass | Charge | Class Name |
|--------------|-------------|----------|------|---------|-------------|

| Phonon Trans-Fast | Fast transverse lattice vibration | 0 | 0 | 0 | `G4PhononTransFast` |
| Phonon Trans-Slow | Slow transverse lattice vibration | 0 | 0 | 0 | `G4PhononTransSlow` |

### Special Simulation Particles

| Particle Name | Description | PDG Code | Mass | Charge | Class Name |
|--------------|-------------|----------|------|---------|-------------|
| Geantino | Non-interacting tracking particle | 0 | 0 | 0 | `G4Geantino` |
| Charged Geantino | Geantino with electric charge | 0 | 0 | +1 | `G4ChargedGeantino` |
| Unknown Particle | Placeholder for undefined particles | 0 | 0 | 0 | `G4UnknownParticle` |
| Charged Unknown | Charged placeholder particle | 0 | 0 | varies | `G4ChargedUnknownParticle` |

## Usage Pattern

All boson classes follow the singleton pattern:

```cpp
// Accessing boson definitions
G4ParticleDefinition* photon = G4Gamma::Definition();
G4ParticleDefinition* optical = G4OpticalPhoton::OpticalPhoton();
G4ParticleDefinition* geantino = G4Geantino::Geantino();

// Alternative access methods
G4Gamma* gamma = G4Gamma::Gamma();
G4OpticalPhoton* opPhoton = G4OpticalPhoton::OpticalPhotonDefinition();

// From particle table
G4ParticleTable* particleTable = G4ParticleTable::GetParticleTable();
G4ParticleDefinition* photon = particleTable->FindParticle("gamma");
G4ParticleDefinition* optical = particleTable->FindParticle("opticalphoton");
```

### Geantino Properties

Geantinos are simulation-only particles that:
- Pass through all materials without interaction
- Record geometry boundaries and volumes traversed
- Can have charge (for electromagnetic field testing)
- Are used for debugging and geometry validation

## Code Examples

### Creating a Photon Source

```cpp
#include "G4ParticleGun.hh"
#include "G4Gamma.hh"

void MyPrimaryGeneratorAction::GeneratePrimaries(G4Event* event)
{
    // Create a gamma ray source
    auto particleGun = new G4ParticleGun(1);
    particleGun->SetParticleDefinition(G4Gamma::Gamma());
    particleGun->SetParticleEnergy(1.*MeV);
    particleGun->SetParticlePosition(G4ThreeVector(0., 0., 0.));
    particleGun->SetParticleMomentumDirection(G4ThreeVector(1., 0., 0.));
    particleGun->GeneratePrimaryVertex(event);
}
```

### Using Optical Photons

```cpp
#include "G4OpticalPhoton.hh"
#include "G4Scintillation.hh"
#include "G4Cerenkov.hh"

void RegisterOpticalPhysics()
{
    // Optical photons are generated by scintillation and Cerenkov processes
    G4ParticleDefinition* opticalPhoton = G4OpticalPhoton::Definition();

    // Enable optical processes
    auto scintillation = new G4Scintillation();
    auto cerenkov = new G4Cerenkov();

    // These processes will generate optical photons
    scintillation->SetTrackSecondariesFirst(true);
    cerenkov->SetMaxNumPhotonsPerStep(300);
}
```

### Tracking Optical Photons

```cpp
#include "G4Track.hh"
#include "G4OpticalPhoton.hh"

void MySteppingAction::UserSteppingAction(const G4Step* step)
{
    G4Track* track = step->GetTrack();

    if (track->GetDefinition() == G4OpticalPhoton::Definition()) {
        // Get optical photon properties
        G4double wavelength = (h_Planck * c_light) / track->GetTotalEnergy();
        G4ThreeVector polarization = track->GetPolarization();

        G4cout << "Optical photon wavelength: "
               << wavelength/nm << " nm" << G4endl;
        G4cout << "Polarization: " << polarization << G4endl;
    }
}
```

### Using Geantinos for Geometry Testing

```cpp
#include "G4Geantino.hh"
#include "G4ChargedGeantino.hh"
#include "G4ParticleGun.hh"

void TestGeometry()
{
    // Geantinos pass through all materials without interaction
    // Useful for checking geometry setup and volume boundaries

    auto gun = new G4ParticleGun(1);
    gun->SetParticleDefinition(G4Geantino::Geantino());
    gun->SetParticleEnergy(1.*GeV);
    gun->SetParticlePosition(G4ThreeVector(0., 0., -10.*m));
    gun->SetParticleMomentumDirection(G4ThreeVector(0., 0., 1.));

    // Track will traverse all volumes without physics interactions
    // Useful for verifying geometry navigation
}

void TestElectromagneticField()
{
    // Charged geantino for testing EM fields without interactions
    auto gun = new G4ParticleGun(1);
    gun->SetParticleDefinition(G4ChargedGeantino::ChargedGeantino());
    gun->SetParticleMomentum(G4ThreeVector(1.*GeV, 0., 0.));

    // Will follow field lines without scattering or energy loss
}
```

### Detecting Photons in Sensitive Detector

```cpp
#include "G4VSensitiveDetector.hh"
#include "G4Gamma.hh"

G4bool MyDetector::ProcessHits(G4Step* step, G4TouchableHistory*)
{
    G4Track* track = step->GetTrack();
    G4ParticleDefinition* particle = track->GetDefinition();

    if (particle == G4Gamma::Definition()) {
        // Photon entered detector
        G4double energy = track->GetKineticEnergy();
        G4ThreeVector position = step->GetPreStepPoint()->GetPosition();

        // Record photon hit
        RecordPhotonHit(energy, position);

        return true;
    }

    return false;
}
```

### Working with Phonons (Cryogenic Detectors)

```cpp
#include "G4PhononLong.hh"
#include "G4PhononTransFast.hh"
#include "G4PhononTransSlow.hh"

void AnalyzePhonons(const G4Step* step)
{
    G4ParticleDefinition* particle = step->GetTrack()->GetDefinition();

    if (particle == G4PhononLong::Definition()) {
        G4cout << "Longitudinal phonon detected" << G4endl;
    }
    else if (particle == G4PhononTransFast::Definition()) {
        G4cout << "Fast transverse phonon detected" << G4endl;
    }
    else if (particle == G4PhononTransSlow::Definition()) {
        G4cout << "Slow transverse phonon detected" << G4endl;
    }
}
```

### Photon Energy and Wavelength Conversion

```cpp
#include "G4Gamma.hh"
#include "G4PhysicalConstants.hh"
#include "G4SystemOfUnits.hh"

void ConvertPhotonProperties()
{
    // Energy to wavelength conversion
    G4double photonEnergy = 2.5 * eV;  // Visible light
    G4double wavelength = (h_Planck * c_light) / photonEnergy;

    G4cout << "Energy: " << photonEnergy/eV << " eV" << G4endl;
    G4cout << "Wavelength: " << wavelength/nm << " nm" << G4endl;

    // Wavelength to energy conversion
    G4double lambda = 500 * nm;  // Green light
    G4double energy = (h_Planck * c_light) / lambda;

    G4cout << "Wavelength: " << lambda/nm << " nm" << G4endl;
    G4cout << "Energy: " << energy/eV << " eV" << G4endl;
}
```

## Physics Background

### Gauge Bosons

Photons are the gauge bosons of quantum electrodynamics (QED), mediating the electromagnetic force:

- **Massless**: Photons travel at the speed of light
- **Spin-1**: Vector boson with two helicity states
- **Stable**: Do not decay in vacuum
- **Self-interaction**: No direct photon-photon coupling in QED

### Photon vs Optical Photon

Geant4 distinguishes between two types of photons:

**G4Gamma (Photon)**:
- High-energy photon (X-rays, gamma rays)
- Undergoes photoelectric effect, Compton scattering, pair production
- Tracked with condensed history Monte Carlo
- PDG code: 22

**G4OpticalPhoton (Optical Photon)**:
- Low-energy photon (UV, visible, IR)
- Undergoes reflection, refraction, absorption, scattering
- Wave optics properties (wavelength, polarization)
- Used in optical physics simulations
- PDG code: -22 (Geant4 convention)

### Optical Processes

Optical photons participate in:
- **Scintillation**: Emission from excited atoms/molecules
- **Cherenkov Radiation**: Emission when charged particle exceeds speed of light in medium
- **Rayleigh Scattering**: Elastic scattering from small particles
- **Boundary Processes**: Reflection, refraction at material interfaces
- **Absorption**: Photon absorbed by material
- **Wavelength Shifting**: Absorption and re-emission at different wavelength

### Phonons in Solid State Physics

Phonons are quantized lattice vibrations in crystals:

- **Longitudinal Phonons**: Atoms oscillate parallel to wave direction
- **Transverse Phonons**: Atoms oscillate perpendicular to wave direction
- **Fast vs Slow**: Different dispersion relations based on crystal symmetry
- Used in cryogenic detector simulations at very low temperatures

### Geantinos - Simulation Tools

Geantinos are not physical particles but simulation utilities:

**Use Cases**:
- Geometry debugging and visualization
- Volume boundary checking
- Navigation algorithm testing
- Electromagnetic field testing (charged geantino)
- Scoring without physics interactions

**Properties**:
- Zero mass and size
- No physics interactions
- Can traverse any material
- Records geometric information
- Useful for detector geometry validation
